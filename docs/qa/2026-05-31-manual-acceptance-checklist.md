# 人人都是程序员 MVP 手动验收清单

## 准备

- 构建产物：`build\windows\x64\runner\Release\everyone_is_programmer.exe`
- 推荐先运行：`flutter test`
- 推荐先构建：`flutter build windows`

## 启动与导航

- [ ] 启动应用后窗口标题显示为 `人人都是程序员`。
- [ ] Windows 任务栏/属性中的产品名称优先显示为 `人人都是程序员`。
- [ ] Windows Release 可执行文件名为 `everyone_is_programmer.exe`。
- [ ] 侧边栏/底部导航只显示三个主区：`资料`、`编程`、`放松`。
- [ ] 默认进入 `资料学习区`。
- [ ] 在横屏和竖屏布局下三大区都能切换。

## 资料学习区

- [ ] 顶部显示 `今日建议路径`。
- [ ] 显示 `学习进度`，初始应为 `0 / 15` 或已保存进度。
- [ ] 资源搜索能按关键词过滤，例如 `RAG`、`MCP`、`推荐算法`。
- [ ] 类型筛选能切换 `视频`、`Skill`、`MCP`、`RAG`、`模型`。
- [ ] 资源卡可以勾选完成，重启后完成状态仍保留。
- [ ] 外部课程资源点击后弹出详情，并可使用 `打开链接`。
- [ ] 本地资源入口可使用 `复制入口`。
- [ ] `本地 RAG 检索` 默认能召回相关内置资料。
- [ ] RAG 检索结果能显示命中的资料分块，例如 `摘要 1` 或 `正文 2`。
- [ ] `RAG 回答草稿` 能根据当前问题显示引用数量和检索依据。
- [ ] `添加资料` 可填写标题、来源、摘要、标签、内容。
- [ ] 添加的 RAG 资料能立即参与检索。
- [ ] 重启后导入的 RAG 资料仍存在。
- [ ] 删除导入资料后，重启后仍保持删除状态。
- [ ] `推荐算法原型` 能按不同学习目标切换推荐结果。

## 编程区

- [ ] 显示 `审计准备进度`、`审计清单`、`本地规则扫描`、`Prompt 模板`。
- [ ] 默认示例代码能扫描出 3 个风险。
- [ ] 修改代码后点击 `运行扫描`，风险列表随代码变化。
- [ ] 高危/中危/低危标签颜色可区分。
- [ ] `复制报告` 能把 Markdown 审计报告复制到剪贴板。
- [ ] `保存报告` 能把 Markdown 审计报告写入本地文件，并显示最近保存路径。
- [ ] Prompt 模板能在三个模板间切换。
- [ ] `复制 Prompt` 能复制当前模板。

## 放松区

- [ ] 显示 `专注节奏`。
- [ ] 可以切换 `专注 25 分钟`、`短休息 5 分钟`、`长休息 15 分钟`。
- [ ] `开始`、`暂停`、`重置` 正常工作。
- [ ] 点击对勾 `记录完成` 后，`节奏记录` 增加一条记录。
- [ ] 记录次数和累计分钟数正确更新。
- [ ] 重启后节奏记录仍存在。
- [ ] 点击 `清空` 后，重启后仍保持清空状态。

## 设置与启动页

- [ ] `界面设置` 中启动界面只提供 `资料`、`编程`、`放松`。
- [ ] `界面设置` 不再显示旧番剧评分相关开关。
- [ ] 选择 `编程` 或 `放松` 为启动页后，重启进入对应区域。
- [ ] 老的 Kazumi 启动页路径会自动归一到 `资料`。
- [ ] 首次启动不再弹出旧插件免责声明、旧 Bangumi 同步失败或旧 Kazumi 快捷方式弹窗。
- [ ] 如启用快捷方式创建能力，桌面快捷方式名称应为 `人人都是程序员`。

## 构建与已知提示

- [ ] `flutter test` 通过。
- [ ] `flutter build windows` 通过。
- [ ] 构建时可能出现 `Nuget is not installed` 和 `webview_windows` CMake warning，目前为既有非致命提示。
- [ ] 并行运行多个 Flutter test 命令时，可能撞到 `build\unit_test_assets` 或 native asset 临时文件锁；顺序重跑即可。
